SDL2 Tutorial in C [02] - Rendering Shapes

May 30, 2022

In this post we will look at the functions for drawing shapes provided by SDL2.

TL;DR#

Skip to the example program.

Overview#

Drawing shapes in SDL2 involves the following steps:

  • Create a renderer
  • Set the drawing color for clearing the screen
  • Clear the screen
  • Set the drawing color for drawing functions
  • Call one of the drawing functions
  • Update the screen

Creating a renderer is usually done once during the initialization of the program. We then use it with the different drawing functions to draw things to the screen.

Note: Creating a renderer is discussed on a previous post, SDL2 Tutorial in C [01] - Creating a Window, Renderer, and Setting an FPS Cap.

Steps 3 and 6 are done once per game loop. At the start of the game loop we clear the screen, and at the end we update the screen.

Steps 4 and 5 can be called multiple times inside the loop. Once the drawing color is set, all the succeeding drawing function calls will be displayed in that color.

Clearing the Screen#

You may think of clearing the screen as getting a new blank piece of paper. We call it at the beginning of our drawing section in the game loop so that every frame we will have a new piece of paper to draw on.

Clearing the screen is done by calling SDL_RenderClear. It returns 0 if successful and a negative value on error.

int SDL_RenderClear(SDL_Renderer * renderer);

Set a Drawing Color#

Setting the drawing color is like selecting the current color of our pencil.
Once we select a color, all the succeeding drawing functions (lines, rect, clear) that we use will be in that color.

To set the drawing color we need to call SDL_SetRenderDrawColor. It returns 0 if successful and a negative value on failure.

int SDL_SetRenderDrawColor(
    SDL_Renderer * renderer, // the renderer
    Uint8 r, // red value 0 - 255
    Uint8 g, // green value 0 - 255
    Uint8 b, // blue value 0 - 255
    Uint8 a  // alpha value 0 - 255
);

Drawing Rectangles#

Drawing a rectangle is done in two steps.

  • Create an SDL_Rect or SDL_FRect struct
  • Pass the created struct to one of the available rectangle drawing function.

The struct for rectangle has two versions, one for int and one for float. They have the same data fields the only difference is the type.

SDL_Rect rect;   // int version
SDL_FRect frect; // float version

// data fields:
// x - x position
// y - y position
// w - width
// h - height

The rectangles can be drawn with or without fill color. Below are the associated drawing functions for rect.

// int version
// draw a rectangle without fill color
int SDL_RenderDrawRect(
    SDL_Renderer * renderer,
    const SDL_Rect * rect
);
// draw a rectangle with fill color
int SDL_RenderFillRect(
    SDL_Renderer * renderer,
    const SDL_Rect * rect
);

//float version
int SDL_RenderDrawRectF(
    SDL_Renderer * renderer,
    const SDL_FRect * rect
);
int SDL_RenderFillRectF(
    SDL_Renderer * renderer,
    const SDL_FRect * rect
);

The functions above returns 0 on success and -1 on error.

Drawing Lines#

Similar to drawing rectangles, drawing lines has an int and float version.

// int version
int SDL_RenderDrawLine(
    SDL_Renderer * renderer,
    int x1, // point 1 x position
    int y1, // point 1 y position
    int x2, // point 2 x position
    int y2  // point 2 y position
);

// float version
int SDL_RenderDrawLineF(
    SDL_Renderer * renderer,
    float x1, 
    float y1, 
    float x2, 
    float y2
);

The usage of the functions above are similar, the only difference is the data type.

Updating the Screen#

SDL2 uses a technique called double buffering for drawing. It means that it has two buffers for drawing. First, is an on-screen buffer which we see on the screen and second, is an off-screen buffer which is updated by our drawing functions. When we are done with the drawing on the current frame we call SDL_RenderPresent to swap the off-screen buffer to the on-screen buffer. This is why we only need to call SDL_RenderPresent only once every frame to update the screen.

void SDL_RenderPresent(SDL_Renderer * renderer);

Coordinate System#

One important thing to note is that in SDL2, the origin (x:0, y:0) of the coordinate system is on the top left corner of the screen.

This means that:

  • adding a positive value to x - moving towards right
  • adding a negative value to x - moving towards left
  • adding a positive value to y - moving downward
  • adding a negative value to y - moving upward

Example Program#

main.c

#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>
#include <SDL2/SDL.h>

int main(int argc, char** args)
{
    const char* title = "Drawing shapes";
    const int window_width = 640;
    const int window_height = 360;
    const float fps_cap = 60.0f;
    const float fps_cap_in_ms = 1000.0f/fps_cap;
    uint64_t start;
    uint64_t end;
    float delta;
    bool is_game_running = true;

    SDL_Window* window;
    SDL_Renderer* renderer;

    SDL_Init(SDL_INIT_EVERYTHING);

    window = SDL_CreateWindow(
        title, 
        SDL_WINDOWPOS_CENTERED,
        SDL_WINDOWPOS_CENTERED,
        window_width,
        window_height,
        0
    );

    renderer = SDL_CreateRenderer(window, -1, SDL_RENDERER_ACCELERATED|SDL_RENDERER_PRESENTVSYNC);

    SDL_Rect red_rect_filled = { .x = 0, .y = 0, .w = 128, .h = 32 };
    SDL_Rect green_rect_filled = { .x = 128, .y = 0, .w = 128, .h = 32 };
    SDL_Rect blue_rect_filled = { .x = 256, .y = 0, .w = 128, .h = 32 };
    SDL_Rect red_rect = { .x = window_width - 128, .y = window_height - 32, .w = 128, .h = 32 };
    SDL_Rect green_rect = { .x = window_width - 256, .y = window_height - 32, .w = 128, .h = 32 };
    SDL_Rect blue_rect = { .x = window_width - 384, .y = window_height - 32, .w = 128, .h = 32 };
    int point1_x = 0;
    int point1_y = 0;
    int point2_x = window_width;
    int point2_y = 0;
    
    while(is_game_running)
    {
        start = SDL_GetPerformanceCounter();
        SDL_Event event;

        while(SDL_PollEvent(&event))
        {
            switch(event.type)
            {
                case SDL_QUIT:
                    is_game_running = false;
                    break;
                default:
                    break;
            }
        }

        // begin update
        point1_y ++;
        if(point1_y > window_height) point1_y = 0;
        
        point2_y ++;
        if(point2_y > window_height) point2_y = 0;
        // end update

        // begin drawing
        SDL_SetRenderDrawColor(renderer, 0, 0, 0, 255);
        SDL_RenderClear(renderer);

        SDL_SetRenderDrawColor(renderer, 255, 0, 0, 255);
        SDL_RenderFillRect(renderer, &red_rect_filled);
        SDL_RenderDrawRect(renderer, &red_rect);

        SDL_SetRenderDrawColor(renderer, 0, 255, 0, 255);
        SDL_RenderFillRect(renderer, &green_rect_filled);
        SDL_RenderDrawRect(renderer, &green_rect);

        SDL_SetRenderDrawColor(renderer, 0, 0, 255, 255);
        SDL_RenderFillRect(renderer, &blue_rect_filled);
        SDL_RenderDrawRect(renderer, &blue_rect);

        SDL_SetRenderDrawColor(renderer, 255, 255, 255, 255);
        SDL_RenderDrawLine(renderer, point1_x, point1_y, point2_x, point2_y);

        SDL_RenderPresent(renderer);
        // end drawing

        end = SDL_GetPerformanceCounter();
        delta = (end - start)/(float)SDL_GetPerformanceFrequency() * 1000.0f;

        if(delta < fps_cap_in_ms)
        {
            SDL_Delay(fps_cap_in_ms - delta);
        }
    }

    SDL_DestroyRenderer(renderer);
    SDL_DestroyWindow(window);
    SDL_Quit();

    return EXIT_SUCCESS;
}
