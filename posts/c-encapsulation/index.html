C Encapsulation and Opaque Types

Encapsulation in most languages is usually done by having a class with a private variable with public methods to access and modify its value.

An Example

Here’s an example in Java for illustrative purposes

// simple example in Java
public class Person {
    // constructor
    public Person() {}

    // private variable accessible only inside the class
    private int age;

    // public methods to interact with its instance
    public int getAge() {
        return this.age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}

If we have an instance of the class above the only way to change the value of age is through its public methods getAge() and setAge().

Now let’s try to do this in C.

Problem

C does not have a concept of a class which bundles data and methods. It also does not have access modifier keywords like public and private.

So, how do we do this?

How to do it in C

Something similar to private variables can be done in C using opaque types. An opaque type is an incomplete type declaration in the header file.

Instead of declaring a struct like this in the header like we usually do

typedef struct person_s 
{
    int age;
} Person;

We will do it like this:

typedef struct person_s Person; // incomplete decalaration or opaque type

Here’s the complete code for the header file.

person.h

#ifndef PERSON_H
#define PERSON_H

typedef struct person_s Person; // opaque type

Person* person_create(); // constructor
void    person_destroy(Person** person);
int     person_get_age(Person* person);
void    person_set_age(Person* person, int age);

#endif

As we can see, even though the declaration for Person is incomplete, we can still use it for our function prototypes.

The details of the struct will be moved to the corresponding implementation file.

Here’s the code for the implementation file.

person.c

#include "person.h"
#include <stdlib.h>

// struct details
// no need for typedef
struct person_s  
{
    int age;
};


// implement functions
Person* person_create()
{
    Person* person = malloc(sizeof(Person));
    return person;
}

void person_destroy(Person** person)
{
    if(*person != NULL) {
        free(*person);
    }
    *person = NULL;
}

int person_get_age(Person* person) 
{
    return person->age;
}

void person_set_age(Person* person, int age) 
{
    (person)->age = age;
}

Let’s try it

Now let’s try to make a program that will use our opaque type.

main.c

#include <stdio.h>
#include "person.h"

int main(int argc, char** args)
{
    Person* person = person_create();
    
    // person->age = 50; <-- will cause compilation error
    // printf("age = %d\n", person->age);  <-- will cause compilation error

    person_set_age(person, 50);
    printf("age = %d\n", person_get_age(person));

    person_destroy(&person);

    return 0;
}

If we try to directly access the age variable inside the struct, our code will not compile. This is because main.c does not know of the struct’s internals. The only way for it to access and modify its values is by using the functions included in the header file.

Encapsulation is a key concept in OOP so most languages that supports this paradigm has a convenient way to do it, but it doesn’t mean that the concept is exclusive to those languages.
