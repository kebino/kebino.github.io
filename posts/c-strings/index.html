Strings in C

May 3, 2022

In C, a string is an array of characters ending with a null terminator ’\0’. If you are coming from a different programming language here’s what you need to know to start working with strings.

Declaration and Initialization#

Let’s start by looking at different ways to declare and initialize a string.

As stated earlier it is an array of characters ending with a null terminator. Here’s how it looks like.

char str_a[6] = { 'h', 'e', 'l', 'l', 'o', '\0' };

I think nobody actually declares them this way in practice but it is important to remember that this is how strings look like in C. Just an array and not an object.

A short hand for the code above is something familiar to a lot of us

char str_a[6] = "hello";

Enclosing characters in double quotes in C is called a string literal. Notice that we created an array with 6 slots even though the word “hello” is 5 letters long. As we learned earlier a string ends with a null terminator so that’s what the extra space is for.

A lot of times you may see strings declared with a char pointer

char* str_a = "hello";

This one looks a lot like how we declare strings in other languages only diffenece is that the data type is a char pointer instead of a string object.

Declaring as a char pointer vs char array, is there a difference?#

Actually there is. Strings declared with a string literal as char pointer is immutable while those declared as char array is mutable.

Here’s an example

char* str_a = "hello";
str_a[0] = 'H'; // Not allowed!

char str_b[6] = "hello";
str_b[0] = 'H'; // This is OK

What’s happenning here?

In str_a, you are actually trying to mutate a string literal which is not allowed. The variable str_a doesn’t have its own copy of the array. It is pointing to the string literal.

Why does it work in str_b’s case?

The line below is actually an array copy of the string.

char str_b[6] = "hello"; // array copy of string

The variable str_b has its own copy of the string so it is not just pointing to the string literal.

If your char pointer is pointing at an array it is ok, but if it is pointing at a string literal like the one above, it is immutable.

String Copy and Assigning Values#

Copying a string to another variable cannot be done by the “=” operator. Trying to do so will just end up with two variables pointing at the same thing.

char str_a[6] = "hello";
char* str_b = str_a; // does not copy the string. It just points to str_a.

str_b[0] = 'H'; // this modifies str_a since str_b is just a pointer to str_a not a copy.

printf("%s\n", str_a); // Hello
printf("%s\n", str_b); // Hello

To actually copy a string, we need to copy it byte by byte. We can do this by iterating to each character and manually copy it to the target variable or we can use strcpy.

Lucky for us, c standard library has the strcpy function for doing this. It takes two arguments, destination and source.

To use it we need to include the string.h header.

Here’s an example usage for strcpy

#include <stdio.h>
#include <string.h>

int main(int argc, char** args)
{
    char str_a[6] = "hello";
    char str_b[6]; // destination must have enough space

    strcpy(str_b, str_a);

    str_b[0] = 'H';

    printf("%s\n", str_a); // hello - unaffected by str_b
    printf("%s\n", str_b); // Hello

    return 0;
}

Comparing strings#

Comparing strings is done by using the functino strcmp in string.h. It returns 0 if the values are equal.

#include <stdio.h>
#include <string.h>

int main(int argc, char** args)
{
    char* str_a = "hello";
    char* str_b = "hello";
    char* str_c = "WORLD";

    int result_ab = strcmp(str_a, str_b); // equal

    if(result_ab == 0) {
        printf("str_a and str_b are equal"); // will be printed
    }

    int result_ac = strcmp(str_a, str_c); // not equal

    if(result_ac == 0) {
        printf("str_a and str_c are equal");  // will not be printed
    }

    return 0;
}

It can be a little confusing at first since 0 also means false and normally if we are comparing, an equal returns true. So keep this in mind when using strcmp, equal values returns 0.

Getting String Length#

Getting string length is provided in the standard library strlen. It also can be found in string.h.

This function takes a string as an argument and returns a size_t as return value.

Example usage of strlen

#include <stdio.h>
#include <string.h>

int main(int argc, char** args)
{
    char* str_a = "hello";

    size_t length = strlen(str_a);

    printf("%lld\n", length); // 5

    return 0;
}

As we can see it returns 5 which is the number of characters in the string.
This means that strlen does not count the ‘\0’ in its return value.

String as a Function Argument#

Example of string as function argument

void greet_a(char* str) {
    printf("%s\n", str);
}

// or

void greet_b(char str[]) {
    printf("%s\n", str);
}

Returning String from a function#

To return a string from a function we need a char pointer as a return value, and in the body of the function allocate memory for the string to be returned.

C memory allocation functions can be found in stdlib.h so we also need to include it.

Example

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

char* return_a_string() {
    // allocate memory with enough space
    // in this example we will return "hello"
    // length of "hello" is 5 + 1 for '\0'
    char* str = malloc(sizeof(char) * 6);
    strcpy(str, "hello");
    return str;
}

int main(int argc, char** args)
{
    char* str = return_a_string();

    printf("%s\n", str); // hello

    free(str); // free allocated memory

    return 0;
}

Why do we need to allocate memory to return a string?#

To better understand let’s have an example of how not to do it

// example of how not to do it
#include <stdio.h>
#include <string.h>

// wrong
char* return_a_string() {
    char str[6] = "hello"; // this is a local variable
    return str; // str will be destroyed
}

int main(int argc, char** args)
{
    char* str = return_a_string(); // return value is destroyed

    printf("%s\n", str); // null

    return 0;
}

In the function body, str is declared as local variable. In C it is often referred to as variables created in the ‘stack’. Variables created in the ‘stack’ are destroyed after it goes out of scope.

For a variable not to be destroyed after it goes out of scope, we need to create it in the ‘heap’ and to create a variable in the ‘heap’ we need to allocate memory.

That’s why the first example worked and the second did not.

Conclusion#

Working with strings in C is a bit different compared to other languages.
In this post we learned the basics to get started working with strings in C.
